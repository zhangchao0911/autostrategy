- 想换标的 → 同时改 `config.yaml`（symbols 部分）和 `data/fetch_data.py`（FETCH_MAP）

## 2. 指标定义

### 指标 1: 平均真实波幅（ATR）

- **用途**：衡量标的波动率，动态计算网格间距
- **数学公式**：TR = max(High-Low, |High-PrevClose|, |Low-PrevClose|)；ATR(N) = SMA(TR, N)
- **参数**：atr_period：默认值 14，合理范围 [7, 28]
- **计算前提**：需要 15 根K线（ATR(14) 需要14根 + 前1根用于计算TR）
- **输出值域**：与价格相同的正数

### 指标 2: 网格中线（基准价格）

- **用途**：网格中心线，作为买卖网格的基准
- **数学公式**：Base = SMA(CLOSE, ma_period)
- **参数**：ma_period：默认值 60，合理范围 [20, 120]
- **计算前提**：需要 60 根K线
- **输出值域**：与价格相同

### 指标 3: 动态网格间距

- **用途**：根据波动率自动调整网格间距
- **数学公式**：GridStep = ATR(14) × grid_multiplier
- **参数**：grid_multiplier：默认值 1.5，合理范围 [0.5, 3.0]
- **计算前提**：依赖 ATR 指标
- **输出值域**：正数

### 指标 4: 网格线序列

- **用途**：以基准价格为中心，向上向下各生成 N 条网格线
- **数学公式**：GridLine(i) = Base + i × GridStep（i 为整数，负数向下）
- **参数**：grid_levels：默认值 5，合理范围 [3, 10]（单侧网格层数）
- **计算前提**：依赖 Base 和 GridStep
- **输出值域**：正数序列

## 3. 信号逻辑

### 3.1 买入信号（BUY）

**触发条件**（所有条件必须同时满足）：

    条件1: 价格跌破某条网格线（Close < GridLine(i) 且上一根 Close >= GridLine(i)）
    条件2: 该网格线对应的持仓为0（同网格线不重复建仓）
    条件3: 当前总持仓数量 < max_grids × 标的数量（不超过总仓位上限）

    附加约束:
    - A股标的（科创50ETF、中证2000ETF）：买入当日不可卖出（T+1规则）
    - 不在涨跌停板时触发（A股检查）
    - 单次买入金额 = 总资金 × single_grid_pct / 标的数量

### 3.2 卖出信号（SELL）

**触发条件**（满足任一即触发）：

    条件1: 价格突破上一条网格线（盈利平仓，Close >= GridLine(i+1)，i为买入网格线编号）
    条件2: 止损：持仓亏损 > stop_loss_pct%
    条件3: A股标的：买入当日不触发卖出信号（T+1规则）

### 3.3 信号优先级

    1. 止损信号 > 网格卖出信号（保命优先）
    2. 买入 vs 卖出冲突时：先执行卖出，清仓后再判断买入
    3. 同一K线内，同一标的只允许一次交易动作
    4. 多标的信号同时触发时，按标的顺序依次执行

## 4. 仓位管理

    方法: 固定比例
    单次买入金额 = 总可用资金 × 5% （每个标的每条网格线的仓位）
    买入数量:
      - A股: floor(买入金额 / 当前价格 / 100) × 100
      - 港股: floor(买入金额 / 当前价格 / 每手股数) × 每手股数（腾讯=100股, 小鹏=100股）
      - 美股: floor(买入金额 / 当前价格)  （不取整，最低1股）

    约束:
    - 单只标的仓位上限: 总资金的 20%
    - 总仓位上限: 总资金的 80%

## 5. 风控规则

> 风控规则是硬约束，任何一条触发都必须执行。
> 默认值已在 Step 4A 中预填，用户可在此基础上修改。

| 规则 | 默认阈值 | 触发动作 |
|------|---------|---------|
| 单笔最大仓位 | 总资金的 5%（每格） | 限制买入数量 |
| 单笔最大亏损 | 买入金额的 5% | 触发止损卖出 |
| 总仓位上限 | 总资金的 80% | 不再开新仓 |
| 最大回撤容忍 | 15% | 全部清仓，暂停交易 5 日 |
| 连续亏损次数 | 3 次 | 降低仓位至 50% |
| 极端行情 | 标的单日跌幅 > 10% | 暂停该标的交易 3 日 |

## 6. 回测参数

> 默认值已在 Step 4A 中按市场类型预填，用户一次性确认。

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 初始资金 | 1,000,000 人民币 | 回测起始资金（港美股按汇率折算） |
| 手续费率 | 见各标的 config | A股万三，港股万五，美股$0.005/股 |
| 印花税率 | A股千一，港美股0 | A股卖出时收取 |
| 滑点 | 0.1% | 买入价+滑点，卖出价-滑点 |
| 回测区间 | 2024-01-01 ~ 2025-12-31 | 覆盖不同市场环境 |

## 7. 禁止事项

> AI 生成代码时**绝对不可**执行以下操作：

1. ❌ 不可使用未来数据（如用当天收盘价做当天交易决策）
2. ❌ 不可忽略手续费和滑点
3. ❌ 不可在信号中引入未在本文档定义的指标
4. ❌ 不可修改信号优先级规则
5. ❌ 不可将风控阈值设为比本文档更宽松的值
6. ❌ A股标的不可违反T+1规则（买入当日不可卖出）
7. ❌ 不可忽略不同市场的交易时间差异

## 8. 已知局限

- 网格策略在单边趋势行情中可能持续买入（下跌趋势）或卖空（上涨趋势，本策略不做空）
- 多标的策略未考虑标的间的相关性，可能存在集中风险
- 港美股标的使用汇率折算，未考虑汇率波动对实际收益的影响
- A股ETF的涨跌停限制（科创50 ±20%，中证2000 ±10%）可能导致极端行情无法平仓
- 美股特斯拉日内波动大，网格间距可能需要更频繁调整
