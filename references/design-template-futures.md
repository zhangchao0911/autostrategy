# 期货策略 — STRATEGY_DESIGN.md 模板

> 本模板用于期货单品种策略（CTA、趋势跟踪、均值回归、套利等）。
> 与股票版模板的核心差异：信号逻辑为4方向（开多/平多/开空/平空）、仓位按保证金计算、需处理合约展期。

生成策略时，将以下内容填充为实际的 `STRATEGY_DESIGN.md` 文件。

---

## 4. 信号逻辑

> 期货信号为4方向：开多（OPEN_LONG）、平多（CLOSE_LONG）、开空（OPEN_SHORT）、平空（CLOSE_SHORT）。
> 必须以**决策树**形式精确描述，不允许使用模糊表述。

### 4.1 开多信号（OPEN_LONG）

**触发条件**（所有条件必须同时满足）：

    条件1: [指标A] [运算符] [阈值]    例: 价格 > MA(20) 且 MA(5) > MA(20)
    条件2: [动量确认]                  例: ATR(14) > ATR(14)的20周期均值
    条件3: [过滤条件]                  例: 成交量 > MA(成交量, 20)

    附加约束:
    - 当前无多头持仓时才触发
    - 不在涨跌停板时触发（期货涨跌停无法成交）

### 4.2 平多信号（CLOSE_LONG）

**触发条件**（满足任一即触发）：

    条件1: [止盈条件]      例: 收益率 > X% 或 价格 < MA(20)
    条件2: [止损条件]      例: 亏损 > 买入金额的 X% 或 价格跌破 [支撑位]
    条件3: [时间止损]      例: 持有超过N根K线且收益 < X%
    条件4: [反向开仓信号]  例: 开空信号触发时，先平多再开空

### 4.3 开空信号（OPEN_SHORT）

**触发条件**（所有条件必须同时满足）：

    条件1: [指标A] [运算符] [阈值]    例: 价格 < MA(20) 且 MA(5) < MA(20)
    条件2: [动量确认]                  例: ATR(14) > ATR(14)的20周期均值
    条件3: [过滤条件]                  例: 成交量 > MA(成交量, 20)

    附加约束:
    - 当前无空头持仓时才触发
    - 不在涨跌停板时触发

### 4.4 平空信号（CLOSE_SHORT）

**触发条件**（满足任一即触发）：

    条件1: [止盈条件]      例: 收益率 > X% 或 价格 > MA(20)
    条件2: [止损条件]      例: 亏损 > 保证金占用金额的 X%
    条件3: [时间止损]      例: 持有超过N根K线且收益 < X%
    条件4: [反向开仓信号]  例: 开多信号触发时，先平空再开多

### 4.5 信号优先级

    1. 止损信号 > 其他平仓信号（保命优先）
    2. 平仓信号 > 开仓信号（先平后开）
    3. 开多 vs 开空冲突时：先平掉反向持仓，再开新仓
    4. 同一K线内最多允许两次交易动作（一次平仓 + 一次开仓）

## 5. 仓位管理

> 期货仓位按**保证金**计算，与股票按金额计算不同。

    保证金比例: [需用户确认，按交易所标准]  # 一手合约占用的保证金 = 合约价值 × 保证金比例
    仓位计算方式: [固定手数 / 固定保证金占用比例 / ATR动态]

    方式1 - 固定保证金比例:
      单笔保证金占用 = 总可用资金 × [需用户确认]
      开仓手数 = floor(单笔保证金占用 / (合约价格 × 合约乘数 × 保证金比例))

    方式2 - ATR动态:
      单笔风险金额 = 总资金 × [需用户确认]
      止损距离 = N × ATR(14)
      开仓手数 = floor(单笔风险金额 / (止损距离 × 合约乘数))

    约束:
    - 单品种最大持仓手数: [需用户确认]
    - 总保证金占用上限: 总资金的 [需用户确认]%
    - 同一方向最大持仓品种数: [需用户确认]

## 6. 风控规则

> 风控规则是硬约束，任何一条触发都必须执行。

| 规则 | 阈值 | 触发动作 |
|------|------|---------|
| 单笔最大保证金占用 | 总资金的 [需用户确认]% | 限制开仓手数 |
| 单笔最大亏损 | 保证金占用金额的 [需用户确认]% | 触发止损平仓 |
| 总保证金占用上限 | 总资金的 [需用户确认]% | 不再开新仓 |
| 最大回撤容忍 | [需用户确认]% | 全部清仓，暂停交易 [需用户确认] 日 |
| 连续亏损次数 | [需用户确认] 次 | 降低仓位至 50%，暂停 [需用户确认] 日 |
| 极端行情 | 涨跌停 / 连续一字板 | 暂停该品种交易 |
| 保证金不足 | 可用资金 < 总资金的 [需用户确认]% | 触发减仓至安全线 |
| 临近交割月 | 距交割 < [需用户确认] 个交易日 | 强制移仓或平仓 |

## 8. 禁止事项

> AI 生成代码时**绝对不可**执行以下操作：

1. ❌ 不可使用未来数据（如用当天收盘价做当天交易决策）
2. ❌ 不可忽略手续费、滑点、保证金占用
3. ❌ 不可在信号中引入未在本文档定义的指标
4. ❌ 不可修改信号优先级规则
5. ❌ 不可将风控阈值设为比本文档更宽松的值
6. ❌ 不可忽略涨跌停板导致的无法成交场景
7. ❌ 不可忽略合约展期成本
